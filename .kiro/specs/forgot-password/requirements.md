# Requirements Document

## Introduction

This document specifies the requirements for a Forgot Password feature in the PeakMetrics ASP.NET Core web application. The feature enables users who have forgotten their password to securely reset it via email verification. The system generates a time-limited password reset token, sends it to the user's registered email address, and allows the user to set a new password using that token.

## Glossary

- **Password_Reset_System**: The subsystem responsible for generating reset tokens, sending reset emails, and processing password reset requests
- **Account_Controller**: The ASP.NET Core MVC controller that handles authentication-related HTTP requests
- **User_Manager**: The ASP.NET Core Identity UserManager service that provides password reset token generation and validation
- **Email_Service**: The MailKit-based service (IEmailService) that sends transactional emails
- **Audit_Log**: The database table that records security-related events for compliance and monitoring
- **Reset_Token**: A cryptographically secure, time-limited token generated by ASP.NET Core Identity for password reset verification
- **Login_Page**: The /Home/Login view where users authenticate
- **Forgot_Password_Page**: The /Account/ForgotPassword view where users request a password reset
- **Reset_Password_Page**: The /Account/ResetPassword view where users set a new password using a valid token

## Requirements

### Requirement 1: Forgot Password Link Display

**User Story:** As a user who has forgotten my password, I want to see a "Forgot Password?" link on the login page, so that I can initiate the password reset process.

#### Acceptance Criteria

1. THE Login_Page SHALL display a "Forgot Password?" link below the Sign In button
2. WHEN a user clicks the "Forgot Password?" link, THE Login_Page SHALL navigate to /Account/ForgotPassword

### Requirement 2: Forgot Password Request Form

**User Story:** As a user who has forgotten my password, I want to submit my email address to request a password reset, so that I can receive instructions to reset my password.

#### Acceptance Criteria

1. THE Forgot_Password_Page SHALL display a form with one input field labeled "Email Address"
2. THE Forgot_Password_Page SHALL display a submit button
3. WHEN the form is submitted with an empty email field, THE Password_Reset_System SHALL display a validation error message "Email is required"
4. WHEN the form is submitted with an invalid email format, THE Password_Reset_System SHALL display a validation error message "Enter a valid email address"

### Requirement 3: Password Reset Token Generation

**User Story:** As a user who has forgotten my password, I want the system to generate a secure reset token when I submit my email, so that only I can reset my password.

#### Acceptance Criteria

1. WHEN a valid email address is submitted, THE Password_Reset_System SHALL query the database for a user with that email address
2. IF the email address exists in the system, THE Password_Reset_System SHALL call userManager.GeneratePasswordResetTokenAsync(user) to generate a Reset_Token
3. IF the email address does not exist in the system, THE Password_Reset_System SHALL display the same success message as if the email existed (to prevent email enumeration attacks)
4. THE Reset_Token SHALL be cryptographically secure and time-limited according to ASP.NET Core Identity default settings

### Requirement 4: Password Reset Email Delivery

**User Story:** As a user who has forgotten my password, I want to receive an email with a password reset link, so that I can securely reset my password.

#### Acceptance Criteria

1. WHEN a Reset_Token is generated for an existing user, THE Email_Service SHALL send an email to the user's registered email address
2. THE email subject SHALL be "Reset your PeakMetrics password"
3. THE email body SHALL contain a button or link with the URL format: /Account/ResetPassword?userId={userId}&token={urlEncodedToken}
4. THE email SHALL be sent using MailKit via the IEmailService interface
5. IF the email fails to send, THE Password_Reset_System SHALL log the error but still display a success message to the user

### Requirement 5: Reset Password Form Display

**User Story:** As a user who has received a password reset email, I want to access a form where I can set a new password, so that I can regain access to my account.

#### Acceptance Criteria

1. WHEN a user navigates to /Account/ResetPassword with valid userId and token query parameters, THE Reset_Password_Page SHALL display a password reset form
2. THE Reset_Password_Page SHALL display two input fields: "New Password" and "Confirm New Password"
3. THE Reset_Password_Page SHALL display a submit button
4. WHEN the userId or token query parameters are missing, THE Reset_Password_Page SHALL display an error message "This reset link is invalid or has expired. Please request a new one."

### Requirement 6: Password Reset Form Validation

**User Story:** As a user resetting my password, I want the system to validate my new password, so that I create a secure password that meets system requirements.

#### Acceptance Criteria

1. WHEN the "New Password" field is empty, THE Password_Reset_System SHALL display a validation error message "Password is required"
2. WHEN the "Confirm New Password" field does not match the "New Password" field, THE Password_Reset_System SHALL display a validation error message "Passwords do not match"
3. WHEN the new password does not meet ASP.NET Core Identity password requirements, THE Password_Reset_System SHALL display the specific validation error returned by Identity (e.g., "Password must be at least 6 characters")

### Requirement 7: Password Reset Execution

**User Story:** As a user resetting my password, I want the system to update my password when I submit a valid new password, so that I can log in with my new credentials.

#### Acceptance Criteria

1. WHEN a valid password reset form is submitted, THE Password_Reset_System SHALL call userManager.ResetPasswordAsync(user, token, newPassword)
2. IF the Reset_Token is valid and not expired, THE Password_Reset_System SHALL update the user's password in the database
3. IF the Reset_Token is invalid or expired, THE Password_Reset_System SHALL display an error message "This reset link is invalid or has expired. Please request a new one."
4. IF the password reset succeeds, THE Password_Reset_System SHALL redirect to /Home/Login with a success toast message "Password reset successfully. Please log in with your new password."

### Requirement 8: Password Reset Audit Logging

**User Story:** As a system administrator, I want all password reset events to be logged, so that I can monitor security events and investigate suspicious activity.

#### Acceptance Criteria

1. WHEN a password reset is successfully completed, THE Password_Reset_System SHALL create a record in the Audit_Log table
2. THE Audit_Log record SHALL have Action = "PasswordReset"
3. THE Audit_Log record SHALL have EntityType = "Auth"
4. THE Audit_Log record SHALL have Details = "{email} reset their password." where {email} is the user's email address
5. THE Audit_Log record SHALL have UserId set to the user's ID
6. THE Audit_Log record SHALL have OccurredAt set to the current UTC timestamp

### Requirement 9: Security and Anti-Enumeration

**User Story:** As a security-conscious system, I want to prevent attackers from discovering which email addresses are registered, so that user privacy is protected.

#### Acceptance Criteria

1. WHEN a password reset is requested for a non-existent email address, THE Password_Reset_System SHALL display the same success message as for an existing email address
2. WHEN a password reset is requested for a non-existent email address, THE Password_Reset_System SHALL NOT send an email
3. THE response time for existing and non-existent email addresses SHALL be similar to prevent timing attacks

### Requirement 10: User Experience and Navigation

**User Story:** As a user, I want clear feedback and navigation throughout the password reset process, so that I understand what to do next at each step.

#### Acceptance Criteria

1. WHEN a password reset email is requested, THE Forgot_Password_Page SHALL display a success message "If your email address is registered, you will receive a password reset link shortly. Please check your inbox."
2. WHEN a password reset is successful, THE Password_Reset_System SHALL redirect to the Login_Page
3. THE success toast message SHALL be displayed on the Login_Page after redirect
4. WHEN a password reset fails due to an invalid or expired token, THE Reset_Password_Page SHALL display an error message with a link to request a new reset
